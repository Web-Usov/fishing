# Доменная модель

Этот документ описывает целевую доменную модель Fishing на уровне продуктового смысла. Здесь фиксируются сущности и их связи, чтобы продуктовые и технические формулировки оставались согласованными по мере развития системы.

## Нотация и терминология

- В продуктовых документах основной термин: `Spot`.
- В техническом контексте может использоваться `FishingPoint` или «точка».
- В рамках модели это одно и то же понятие: выбранная пользователем географическая точка для прогноза и рыболовного контекста.

## Сущности

### Waterbody

**Назначение:** описывает водоём как устойчивый природный объект, в пределах которого существует набор пользовательских точек рыбалки.

**Атрибуты верхнего уровня:**

- человекочитаемое название;
- тип водоёма (река, озеро, водохранилище и т.д.);
- географический контур или область присутствия;
- базовые характеристики окружения, важные для интерпретации рыбалки.

**Ключевые связи:**

- `Waterbody` включает множество `Spot`;
- `Trip/Session` может быть привязана к водоёму напрямую, если пользователь фиксирует выезд на уровень «поехал на водоём», без точного выбора точки.

### Spot

**Назначение:** конкретная точка рыбалки, вокруг которой строится основной пользовательский цикл выбора места и получения прогноза.

**Атрибуты верхнего уровня:**

- координаты (`lat`, `lng`);
- пользовательское имя или метка;
- контекст точки (тип берега, глубина, заметки);
- привязка к водоёму при наличии этой информации.

**Ключевые связи:**

- `Spot` может принадлежать одному `Waterbody`;
- у `Spot` может быть много `Forecast` по датам;
- `Trip/Session` обычно привязывается к одному `Spot`;
- `Catch` фиксируется в контексте `Spot` через конкретную сессию.

### Forecast

**Назначение:** прогнозная оценка условий и вероятности клева для выбранной точки на конкретную дату.

**Атрибуты верхнего уровня:**

- дата прогноза;
- интегральная оценка клева;
- факторы, влияющие на оценку (погодный и контекстный слой);
- локальное суточное окно точки (расчёт через `timestamp + timezone`);
- статус точности, например normal или fallback.

**Ключевые связи:**

- `Forecast` относится к одному `Spot`;
- `Forecast` используется пользователем как вход в решение о планировании `Trip/Session`.

### Trip/Session

**Назначение:** факт или план выезда пользователя на рыбалку в конкретное время и место.

**Атрибуты верхнего уровня:**

- статус (план, в процессе, завершено, отменено);
- временной интервал выезда;
- условия и заметки пользователя;
- итоговая оценка выезда и наблюдения.

**Ключевые связи:**

- `Trip/Session` принадлежит пользователю;
- `Trip/Session` связывается с одним `Spot` и опционально с `Waterbody`;
- в рамках одной `Trip/Session` может быть зафиксировано много `Catch`;
- `Trip/Session` может ссылаться на `Forecast`, который использовался при принятии решения.

### Catch

**Назначение:** запись о конкретном улове в рамках выезда, которая формирует персональную историю и материал для сравнения прогноза с фактом.

**Атрибуты верхнего уровня:**

- время фиксации;
- вид рыбы и оценка размера/веса;
- способ ловли, приманка и дополнительные заметки;
- опциональные медиа и контекст наблюдений.

**Ключевые связи:**

- `Catch` принадлежит одной `Trip/Session`;
- через `Trip/Session` наследует контекст `Spot` и, при наличии, `Waterbody`;
- набор `Catch` используется для последующего анализа качества прогнозной модели.

## Карта отношений и ограничения

- `Waterbody 1 -> N Spot`.
- `Spot 1 -> N Forecast`.
- `User 1 -> N Trip/Session`.
- `Trip/Session 1 -> N Catch`.
- `Trip/Session N -> 1 Spot` как базовый сценарий, `Trip/Session N -> 1 Waterbody` как опциональная привязка уровня водоёма.

Явные ограничения модели:

- `Forecast` уникально связан с парой `(Spot, date)`;
- `Catch` не существует вне `Trip/Session`;
- `Trip/Session` должен иметь привязку к месту рыбалки минимум на одном уровне: `Spot` или `Waterbody`;
- если у `Trip/Session` задан `Spot`, то связанный `Waterbody` должен быть согласован с этим `Spot`, когда такая связь известна.

## Как это соотносится с текущим MVP

Текущий MVP реализует только coordinate-first уровень `Spot` (точка по координатам) и `Forecast` для этой точки. Сущности `Trip/Session`, `Catch` и `Waterbody` в текущей версии являются целевой доменной моделью и документируются как следующий слой развития, а не как уже поставленное поведение.

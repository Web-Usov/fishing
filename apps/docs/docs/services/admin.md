# Admin

## Назначение

`apps/admin` — административная поверхность продукта. Сейчас это минимальная оболочка, которая резервирует отдельный контур под внутренние и backoffice-сценарии.

## Для чего нужен admin

- управление доменными справочниками;
- контроль качества данных;
- внутренние операционные задачи;
- будущая модерация пользовательского контента.

## Текущее состояние

- есть отдельное приложение в монорепозитории;
- есть отдельный runtime и собственный UI-слой;
- полноценные административные workflows ещё не реализованы.

## Стек

- `Next.js`
- `React`
- `TypeScript`
- `@fishing/ui`

## Принципы

- admin не должен быть копией публичного web;
- это отдельный внутренний контур со своей информационной архитектурой;
- все будущие административные возможности должны проходить через отдельную документацию и backlog-декомпозицию.

## Инфраструктура и качество

- сервис `admin` в Docker Compose;
- обязательные build/typecheck;
- в будущем — отдельные CRUD/test сценарии и операционные runbooks.

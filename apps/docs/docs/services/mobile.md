# Mobile

## Назначение

`apps/mobile` — мобильная поверхность приложения на Expo/React Native. Сейчас это подготовленная платформа для следующего этапа, а не полная реализация основного пользовательского сценария.

## Доменный смысл и зона ответственности

Mobile должен стать главным интерфейсом использования приложения «в поле» — на выезде, у водоёма и при фиксации фактического результата.

## Входы и выходы

- **Входы:** пользовательские действия в полевых сценариях, API-данные прогноза и будущие данные журнала улова.
- **Выходы:** мобильный UX core loop, согласованный по смыслу с web, но platform-native по взаимодействию.
- Текущий статус: инфраструктурная и shared-база готова, полный прогнозный сценарий ещё в разработке.

## Текущее состояние

- есть Expo shell;
- приложение включено в монорепозиторий;
- shared UI и shared packages уже доступны;
- полный прогнозный сценарий пока не доведён до product-ready уровня.

## Стек

- `Expo`
- `React Native`
- `TypeScript`
- `@fishing/shared-ui`

## Зависимости

- `apps/api` (серверные сценарии);
- shared packages: домен, контракты и UI-примитивы;
- корректная Metro/workspace-конфигурация монорепозитория.

## Принципы развития

- смысл пользовательского сценария должен совпадать с web;
- mobile UX не обязан быть визуально идентичен web;
- shared следует использовать там, где он уменьшает стоимость развития, а не там, где он ломает платформенную логику.

## Инфраструктура

- сервис `mobile` в Docker Compose;
- Expo LAN flow;
- зависимость от корректной workspace/Metro-конфигурации.

## Качество

- текущий baseline: build/typecheck shell;
- дальнейшая цель — screen/integration тесты и полный пользовательский сценарий прогноза.

## Типовые сценарии изменений

- Добавить screen сценария прогноза: опираться на shared-контракты и platform-native взаимодействие.
- Перенести логику из web: сначала проверить границы shared-vs-platform, затем адаптировать UX под mobile.
- Изменить data/state слой: синхронизировать hooks/клиент с API-контрактами и обновить integration-проверки.

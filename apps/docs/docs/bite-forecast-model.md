# Модель прогноза клёва

## Назначение документа

Этот документ описывает текущую модель прогноза клёва, которая используется для 7-дневного прогноза по выбранной точке. Описание синхронизировано с реализацией в `packages/domain-bite-forecast/src/index.ts` и фиксирует, как модель работает прямо сейчас.

Модель является статической экспертной формулой. Она не обучается на исторических данных и не выполняет региональную калибровку. Внутри расчёта используется hourly-first подход с дневной агрегацией.

## Входные данные модели

На вход передаются:

- координата точки (`point`);
- время расчёта (`timestamp`) и таймзона (`timezone`);
- погодный снимок (`weather`):
  - `pressureHpa`;
  - `airTemperatureC`;
  - `windSpeedMps`;
  - `cloudCoverPct`;
  - `precipitationMm`.
- опционально почасовая серия (`hourlyWeather`) с шагом 1–3 часа:
  - `timestamp`;
  - `pressureHpa`, `airTemperatureC`, `windSpeedMps`, `cloudCoverPct`, `precipitationMm`;
  - `windDirectionDeg?`, `precipitationProbabilityPct?`.
- опционально `locale` (`ru|en`) и `debug`-флаг.

Формат и диапазоны валидации входа задаются схемами в `packages/shared-zod/src/index.ts`.

## Слои модели и факторы

Модель считается по часам и затем агрегируется в дневной индекс.

### Почасовой индекс

Для каждого часа считается:

- погодный подиндекс `I_meteo`;
- множитель солунарного окна `K_sol`;
- множитель светового окна `K_light`;
- сезонный множитель `K_season`.

Итог часа: `I_hour = clip(I_meteo * K_sol * K_light * K_season, 0..100)`.

### Погодный подиндекс (`I_meteo`)

Используются факторы:

### Давление (`pressureHpa`)

- `1008..1022` -> `+12`
- иначе -> `-10`

### Ветер (`windSpeedMps`)

- `<= 3` -> `+10`
- `<= 7` -> `+2`
- иначе -> `-12`

### Температура воздуха (`airTemperatureC`)

- `12..22` -> `+11`
- `5..28` -> `+3`
- иначе -> `-14`

### Облачность (`cloudCoverPct`)

- `20..70` -> `+6`
- иначе -> `0`

### Осадки (`precipitationMm`)

- `0` -> `+4`
- `< 2` -> `+1`
- иначе -> `-8`

### Солунарный и световой слои (`timestamp` + `timezone`)

Локальное время точки рассчитывается через комбинацию `timestamp` и `timezone`.

- солунарные окна добавляют повышающие/понижающие множители в зависимости от локального часа;
- окна рассвета/заката дают отдельный повышающий множитель активности;
- для UI/debug в hourly-выходе дополнительно проставляются теги (`solunar`, `sunrise`, `sunset`, `stable_pressure`, `favorable_wind`).

### Сезонность (`timestamp`)

Сезон определяется по месяцу из `timestamp` (северное полушарие):

- весна (3..5) -> `+4`
- лето (6..8) -> `+1`
- осень (9..11) -> `+4`
- зима (12, 1, 2) -> `-3`

## Как считается итоговый score/level/confidence

1. Считаются 24 почасовых индекса `I_hour`.
2. Вычисляются:
   - `I_day_max` — максимум по суткам;
   - `I_day_mean(>=60)` — среднее по часам выше порога «рабочей активности».
3. Дневной индекс: `I_day = round(0.7 * I_day_max + 0.3 * I_day_mean)`.
4. Финальный score: `clamp(I_day, 0..100)`.

Далее score переводится в уровень:

- `excellent`, если `score >= 80`
- `good`, если `score >= 60`
- `moderate`, если `score >= 40`
- `poor` в остальных случаях

Далее считается confidence:

- учитывается смещение `offset = abs(score - 50)` и качество/полнота входных данных;
- `high`, если `offset >= 25`
- `medium`, если `offset >= 15`
- `low` в остальных случаях

Отдельно в ответе выбирается самый сильный фактор по модулю `impact`, и на его основе формируется короткое текстовое объяснение.
Пояснение возвращается в двух формах:

- агрегированная строка `explanation` (под выбранный locale);
- структурный список `factors[]` для UI.

При `debug=true` дополнительно возвращается `expanded`:

- `hourly[24]`;
- `bestWindows[]` (`from`, `to`, `peakScore`, `tags`);
- `debugBreakdown[]` по коэффициентам.

Пример explainability-строки:

- `Хороший клёв: ключевой фактор — утренний выход рыбы.`

## Ограничения текущей модели

Текущая версия сделана как MVP-слой для быстрого и понятного 7-дневного прогноза. Важно учитывать ограничения:

- нет дифференциации по видам рыб;
- `water_type` сознательно не участвует в формуле текущего этапа;
- нет учёта глубины, рельефа и структуры точки;
- используется статическая экспертная формула, модель не обучена на данных и не регионально откалибрована.

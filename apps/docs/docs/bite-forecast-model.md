# Модель прогноза клёва

## Назначение документа

Этот документ описывает текущую MVP-модель прогноза клёва, которая используется для 7-дневного прогноза по выбранной точке. Описание синхронизировано с реальной реализацией в `packages/domain-bite-forecast/src/index.ts` и фиксирует, как модель работает прямо сейчас.

Модель является статической экспертной формулой. Она не обучается на исторических данных и не выполняет региональную калибровку.

## Входные данные модели

На вход передаются:

- координата точки (`point`);
- время расчёта (`timestamp`) и таймзона (`timezone`);
- тип водоёма (`waterbodyType`): `lake`, `river`, `reservoir`, `pond`;
- погодный снимок (`weather`):
  - `pressureHpa`;
  - `airTemperatureC`;
  - `windSpeedMps`;
  - `cloudCoverPct`;
  - `precipitationMm`;
  - `moonIlluminationPct`.

Формат и диапазоны валидации входа задаются схемами в `packages/shared-zod/src/index.ts`.

## Факторы и их влияние

Итог строится как сумма вкладов семи факторов.

### Давление (`pressureHpa`)

- `1008..1022` -> `+12`
- иначе -> `-10`

### Ветер (`windSpeedMps`)

- `<= 3` -> `+10`
- `<= 7` -> `+2`
- иначе -> `-12`

### Температура воздуха (`airTemperatureC`)

- `12..22` -> `+11`
- `5..28` -> `+3`
- иначе -> `-14`

### Облачность (`cloudCoverPct`)

- `20..70` -> `+6`
- иначе -> `0`

### Осадки (`precipitationMm`)

- `0` -> `+4`
- `< 2` -> `+1`
- иначе -> `-8`

### Лунная освещённость (`moonIlluminationPct`)

- `25..75` -> `+5`
- иначе -> `-3`

### Тип водоёма (`waterbodyType`)

- `lake` -> `+5`
- `reservoir` -> `+3`
- `pond` -> `+2`
- `river` -> `0`

## Как считается итоговый score/level/confidence

1. Базовый уровень: `50`.
2. Складываются влияния всех факторов.
3. Считается сырое значение: `50 + sum(impacts)`.
4. Применяется формула score: `clamp(round(50 + sum(impacts)), 0..100)`.

Далее score переводится в уровень:

- `excellent`, если `score >= 80`
- `good`, если `score >= 60`
- `moderate`, если `score >= 40`
- `poor` в остальных случаях

Далее считается confidence:

- `high`, если `score >= 70` или `score <= 30`
- `medium`, если `score >= 45`
- `low` в остальных случаях

Отдельно в ответе выбирается самый сильный фактор по модулю `impact`, и на его основе формируется короткое текстовое объяснение.

## Ограничения текущей модели

Текущая версия сделана как MVP-слой для быстрого и понятного 7-дневного прогноза. Важно учитывать ограничения:

- нет дифференциации по видам рыб;
- нет сезонной логики и логики времени года;
- нет учёта глубины, рельефа и структуры точки, кроме грубого типа водоёма;
- используется статическая экспертная формула, модель не обучена на данных и не регионально откалибрована.

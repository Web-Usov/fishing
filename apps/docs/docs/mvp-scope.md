# Объём MVP

Этот документ отвечает на вопрос: что уже входит в MVP, что находится за его текущей границей и какой продукт мы реально поставляем прямо сейчас.

Документ ориентирован прежде всего на **Product-режим**, но используется и разработкой как операционная граница реализации.

## Что уже входит в MVP

- Реальная web-карта (Яндекс Карты через adapter layer).
- Coordinate-first сценарий: пользователь кликает любую точку.
- 7-дневный прогноз клёва для выбранной координаты.
- Реальный погодный контекст через Open-Meteo без synthetic-подстановок в расчёте прогноза.
- Same-origin weather endpoint `/api/weather/forecast` для web.
- Детальная сводка по дням и факторам прогноза.
- URL-state через `?lat=...&lng=...`.
- Плавное обновление прогноза без сброса предыдущих данных.
- Empty/loading/error состояния и retry-сценарий.
- Переключение theme и locale.
- Геолокация в web и базовый runtime UX.
- Документационный и агентский контур в `apps/docs`.

## Что MVP уже проверяет

MVP проверяет не «весь рыболовный продукт», а конкретный core loop:

1. Пользователь открывает карту.
2. Выбирает точку.
3. Получает прогноз.
4. Понимает объяснение по факторам.
5. Принимает решение о выезде.

## Что сознательно ещё не входит в MVP

- социальная лента / отчёты / рекомендации;
- полноценный журнал улова;
- production-grade ingestion pipeline по погоде и внешним данным;
- развитая система пользователей и доступа;
- зрелые административные workflows;
- полноценный mobile parity по основному сценарию.

## Эксплуатационные заметки MVP

- Основной локальный режим запуска — Docker Compose из корня.
- Использование по LAN поддерживается.
- Для web обязательна hydration-safe логика там, где используются browser-only значения.
- При недоступности внешнего погодного источника UI должен явно показывать ошибку недоступности данных и не строить псевдопрогноз.

## Что является следующим шагом после MVP

- стабилизация качества прогноза и explainability;
- перенос полного сценария в mobile;
- журнал улова;
- пользовательские точки, история и персональный контекст;
- подготовка social/community слоя.

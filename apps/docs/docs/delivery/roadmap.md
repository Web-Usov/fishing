# Roadmap

Roadmap фиксирует крупные этапы развития продукта. Это стратегический документ: он показывает последовательность больших блоков, а не заменяет backlog.

**Для кого:** Product + Develop.

**Навигация внутри delivery:** [Backlog](backlog.md) · [Task State](task-state.md)

## Этап 1. Map-first MVP

Цель: дать пользователю рабочий сценарий выбора точки и получения прогноза клёва.

## Этап 2. Стабилизация прогнозного продукта

Цель: повысить качество, объяснимость и устойчивость прогнозного сценария на web и подготовить mobile-перенос.

## Этап 3. Журнал улова

Цель: превратить разовую проверку прогноза в накопление личной истории рыбалки.

## Этап 4. Пользовательский контекст и точки

Цель: дать историю точек, избранное, персональные сценарии повторного использования.

## Этап 5. Социальный слой

Цель: отчёты, посты, рекомендации, коллективный рыболовный контекст.

## Этап 6. Production readiness

Цель: довести систему до зрелого operational уровня.

## Как использовать roadmap

- roadmap отвечает за большие этапы;
- для разбиения на конкретные блоки использовать [Backlog](backlog.md);
- для текущего статуса — [Task State](task-state.md).
